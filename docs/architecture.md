# 아키텍처 결정 기록

## 시스템 개요

한국어-영어 실시간 대화 번역 앱. 두 사람이 각자의 언어로 말하면 상대방 언어로 번역해주고, AI 에이전트가 대화 내용을 분석해 팩트체크 정보를 비동기로 제공한다.

## 기술 스택

| 레이어 | 기술 |
|--------|------|
| 프론트엔드 | React 19, TypeScript, Vite |
| 백엔드 | Python, FastAPI |
| 번역 | DeepL REST API |
| 에이전트 | Claude API (tool use) + 웹 검색 도구 |

## 핵심 아키텍처 결정

### 번역과 에이전트 분리

번역과 에이전트는 완전히 독립적인 서비스로 동작한다. 프론트엔드는 두 요청을 동시에 발행하며, 어느 쪽도 상대를 기다리지 않는다.

```
사용자 입력
    ├── POST /api/translate  →  DeepL  →  번역 결과 즉시 표시
    └── POST /api/agent/stream  →  Claude  →  SSE로 팩트체크 비동기 도착
```

### 번역 방식: REST

DeepL API는 짧은 대화문 기준 응답 시간이 200~500ms로 빠르기 때문에 REST로 충분하다. 로딩 인디케이터로 대기 UX 처리.

### 에이전트 응답 방식: SSE (Server-Sent Events)

에이전트는 웹 검색 도구를 사용하므로 응답 시간이 불규칙하다. SSE로 스트리밍하면 "분석 중" 상태를 실시간으로 표시할 수 있어 UX가 자연스럽다. 클라이언트→서버 단방향이므로 WebSocket보다 SSE가 적합하다.

### 에이전트 트리거: 항상 실행

모든 번역 발생 시 에이전트를 항상 실행한다. 팩트체크 필요 여부는 Claude가 `tool_choice: auto` 설정으로 스스로 판단한다. 팩트체크할 내용이 없으면 웹 검색 없이 스트림을 종료한다.

별도의 "필요 여부 판별" 로직을 두지 않는다. 판별 자체도 LLM 호출이 필요하므로 단순화를 위해 Claude에게 전적으로 위임한다.

### 지원 언어

현재 한국어(KO) ↔ 영어(EN) 양방향만 지원한다.
