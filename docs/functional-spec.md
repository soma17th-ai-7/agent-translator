# 기능 명세

## 1. 대화 흐름

두 사람이 하나의 기기를 사이에 두고 대화한다. 하단 패널은 기기를 정방향으로 보는 화자, 상단 패널은 180도 회전된 방향으로 보는 화자가 사용한다.

```
[1] 하단 화자 입력 (음성 또는 텍스트)
[2] POST /api/translate + POST /api/agent/stream 동시 발행
[3] 번역 결과 → 상단 패널에 표시 (상대방이 읽을 수 있도록)
[4] 에이전트 팩트체크 결과 → 하단 패널에 비동기 오버레이 표시
[5] 상단 화자 응답 입력
[6] POST /api/translate + POST /api/agent/stream 동시 발행
[7] 번역 결과 → 하단 패널에 표시
[8] 에이전트 팩트체크 결과 → 하단 패널에 비동기 오버레이 표시
[계속 반복 — 순서 제약 없음]
```

두 화자는 순서 제약 없이 자유롭게 발화할 수 있다. 번역 중에는 해당 패널의 입력만 비활성화된다.

## 2. 언어 설정

### 언어 교환 버튼
- 두 패널 사이 구분선 중앙에 배치
- 클릭 시 양쪽 패널의 언어가 즉시 교환되고 대화 기록 초기화

### 언어 선택 드롭다운
- 각 패널의 언어 레이블을 클릭하면 한국어 / English 선택 목록 표시
- 선택 시 언어 교환 버튼과 동일하게 동작
- 현재 한국어 ↔ 영어만 지원

## 3. 입력 방식

각 패널은 음성 입력과 텍스트 입력을 지원하며, 패널별로 독립적으로 전환 가능하다.

### 음성 입력
- 마이크 버튼을 누르면 청취 시작 (버튼 pulse 애니메이션)
- 현재: STT 미구현, 하드코딩된 샘플 텍스트로 시뮬레이션

### 텍스트 입력
- 키보드 아이콘을 눌러 모드 전환
- 입력창에 텍스트 작성 후 Enter 또는 전송 버튼으로 제출
- 제출된 텍스트가 해당 패널의 발화 텍스트로 표시됨

## 4. 번역 표시

각 발화의 최신 상태를 기준으로 표시한다.

| 상황 | 하단 패널 | 상단 패널 |
|------|-----------|-----------|
| 하단 화자 발화 직후 | 원문(대) + 번역 확인(소) | 번역된 텍스트(대) |
| 상단 화자 응답 후 | 원문(대) + 상대 응답 번역(대, 회색) | 응답 원문(대) + 번역 확인(소) |
| 번역 대기 중 | 로딩 인디케이터 | 로딩 인디케이터 |

텍스트는 `fadeUp` 애니메이션으로 등장한다.

## 5. 에이전트 팩트체크

### 동작 방식
- 번역 요청과 동시에 에이전트 스트림 요청 발행 (독립 비동기)
- 에이전트는 **누적 대화 히스토리 전체**를 컨텍스트로 받아 최신 발화를 기준으로 팩트체크 판단
- 사용자의 현재 위치 정보(역지오코딩 결과)도 컨텍스트로 전달
- 팩트체크 필요 시: 결과를 하단 패널 오버레이에 표시 (120자 이내)
- 팩트체크 불필요 시: 스트림 즉시 종료 (UI 변화 없음)
- 이전 에이전트 응답은 `Message.agentResponse`에 기록되어 다음 호출 시 멀티턴 구조로 재사용 (KV 캐시 활용)

### UI 표시
- `status: searching` 수신 시: 스피너 아이콘과 함께 오버레이 등장
- `result` 수신 시: 팩트체크 텍스트 표시
- 완료 후 X 버튼으로 닫기 가능
- 새 메시지 전송 시 이전 오버레이 자동 닫힘

### 결과 언어
팩트체크 결과는 항상 **하단 패널 화자의 언어**(`response_lang`)로 표시된다.
어느 화자가 발화했든 관계없이 동일한 언어로 표시된다.

### 위치 컨텍스트
- 앱 로드 시 브라우저 Geolocation API로 위치 권한 요청
- 허용 시 Nominatim(OpenStreetMap)으로 역지오코딩 → `"강남구, 서울, 대한민국"` 형태
- 두 패널 사이 구분선 좌측에 위치 아이콘과 함께 표시
- 에이전트 요청마다 포함되어 현지 요금·거리 등 맥락 있는 팩트체크 가능

## 6. 디버그 모드

- 두 패널 사이 구분선 우측 🐛 버튼으로 토글
- 활성화 시: 에이전트의 팩트체크 결정 근거(한국어)를 앱 우측 사이드바에 표시
- 팩트체크 수행/불필요 여부와 이유를 실시간으로 확인 가능
- 프로덕션 사용 시에는 비활성화

## 7. 에러 처리

### 번역 실패
- API 오류 발생 시 번역이 표시될 위치에 에러 메시지와 재시도 버튼 표시
- 에러 메시지는 해당 패널의 언어로 표시 (한국어 패널 → "번역에 실패했습니다.", 영어 패널 → "Translation failed.")
- 재시도 버튼 클릭 시 에러 상태 초기화, 해당 화자가 다시 입력 가능한 상태로 복귀

### 에이전트 오류
- 에이전트 스트림 중 오류 발생 시 오버레이 미표시, 콘솔에 에러 로그
- 오류 발생 시에도 번역 결과와 대화 흐름에는 영향 없음

### 위치 권한 거부
- 브라우저에서 위치 권한을 거부하면 위치 컨텍스트 없이 정상 동작
- 위치 뱃지가 표시되지 않으며 에이전트는 위치 정보 없이 팩트체크 수행

## 8. 미구현 기능 (프로토타입 범위 외)

- 실제 STT(음성 인식): 현재 setTimeout으로 시뮬레이션
- 다국어 확장 (현재 KO ↔ EN만 지원)
- 사용자 인증
- 세션 저장 및 대화 이력 복원
