# 기능 명세

## 1. 대화 흐름

두 사람이 하나의 기기를 사이에 두고 대화한다. 하단 패널은 한국어 화자, 상단 패널은 영어 화자가 사용한다. 상단 패널은 180도 회전하여 반대편에서 읽기 편하게 표시된다.

```
[1] 한국어 화자 입력 (음성 또는 텍스트)
[2] POST /api/translate (KO → EN) + POST /api/agent/stream 동시 발행
[3] 영어 번역 결과 → 상단 패널에 표시
[4] 에이전트 팩트체크 → 도착하는 대로 비동기 표시 (하단 패널 오버레이)
[5] 영어 화자 입력 (음성 또는 텍스트)
[6] POST /api/translate (EN → KO) + POST /api/agent/stream 동시 발행
[7] 한국어 번역 결과 → 하단 패널에 표시
[8] 에이전트 팩트체크 → 도착하는 대로 비동기 표시
```

## 2. 입력 방식

각 패널은 음성 입력과 텍스트 입력을 지원하며, 패널별로 독립적으로 전환 가능하다.

### 음성 입력
- 마이크 버튼을 누르면 청취 시작 (버튼 pulse 애니메이션)
- 발화 완료 후 자동으로 번역 요청 발행
- 현재 프로토타입: STT 미구현, 하드코딩된 샘플 텍스트 사용

### 텍스트 입력
- 키보드 아이콘을 눌러 모드 전환
- 입력창에 텍스트 작성 후 Enter 또는 전송 버튼으로 제출
- 제출된 텍스트가 해당 패널의 발화 텍스트로 표시됨

## 3. 번역 표시

번역 결과는 두 패널에 각각 다르게 표시된다.

| 패널 | 표시 내용 |
|------|-----------|
| 화자 패널 (원문) | 본인이 말한 텍스트 (대형, 굵게) + 번역된 텍스트 (소형, 회색) |
| 상대방 패널 (번역) | 번역된 텍스트 (대형, 굵게) |

텍스트는 `fadeUp` 애니메이션으로 등장한다.

## 4. 에이전트 팩트체크

### 동작 방식
- 번역 요청과 동시에 에이전트 스트림 요청 발행
- Claude가 대화 내용을 분석해 팩트체크 필요 여부를 자체 판단
- 팩트체크할 내용이 있으면 웹 검색 수행 후 결과 스트리밍
- 팩트체크할 내용이 없으면 스트림 즉시 종료 (UI 변화 없음)

### UI 표시
- 팩트체크 결과가 있을 경우: 화자 패널 하단에 반투명 오버레이 카드로 표시
- SSE `status: searching` 이벤트 수신 시: 오버레이에 "검색 중..." 인디케이터 표시
- SSE `result` 이벤트 수신 시: 텍스트를 스트리밍으로 표시 (타이핑 효과)
- 카드를 탭하면 닫힘

### 결과 언어
팩트체크 결과는 해당 발화를 한 화자의 언어로 표시된다.
- 한국어 화자의 발화 → 팩트체크 결과를 한국어로 표시
- 영어 화자의 발화 → 팩트체크 결과를 영어로 표시

### 에이전트에 전달되는 컨텍스트
- 화자의 원문 텍스트
- 번역된 텍스트
- 원문/번역 언어 정보

## 5. 상태 관리

대화는 단계(`step`)로 관리된다.

| step | 상태 |
|------|------|
| 0 | 초기 대기 (한국어 화자 입력 가능) |
| 1 | 한국어 화자 발화 완료 |
| 2 | 영어 번역 표시 완료 (영어 화자 입력 가능) |
| 3 | 영어 화자 발화 완료 |
| 4 | 한국어 역번역 표시 완료 |

에이전트 팩트체크 표시 여부는 step과 독립적으로 관리된다.

대화를 다시 시작하려면 에이전트 카드를 탭하거나 별도 리셋 동작을 수행한다.

## 6. 미구현 기능 (프로토타입 범위 외)

- 실제 STT(음성 인식): 현재 setTimeout으로 시뮬레이션
- 세션/대화 이력 관리
- 다국어 확장 (현재 KO ↔ EN만 지원)
- 사용자 인증
